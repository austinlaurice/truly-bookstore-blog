# 真的書店 Blog 發文指南

這份指南寫給**不寫程式的夥伴**。你只需要一個瀏覽器和 GitHub 帳號（需要被加入這個 repo 的協作者）。

## 核心概念

- **一篇文章 = 一個文字檔案**，放在 `_posts/` 資料夾裡
- 按下 Commit（提交）就等於發佈，**約 1–2 分鐘後上線**到 blog.in-common.tw
- 檔案用 Markdown 格式寫——就是純文字加上少量符號，下面會教

---

## 新增一篇文章（完整步驟）

### 第 1 步：建立檔案

1. 打開這個 repo 的 [`_posts/` 資料夾](../../tree/main/_posts)
2. 點右上角 **Add file → Create new file**
3. 檔名格式**必須**是：`年-月-日-英文短名.md`
   - ✅ 正確：`2026-07-15-summer-picks.md`
   - ❌ 錯誤：`夏日選書.md`（沒有日期）、`2026-7-15-picks.md`（月份要兩位數）
   - 英文短名會變成文章網址的一部分，用小寫英文加連字號

### 第 2 步：貼上範本並填寫

打開 repo 根目錄的 [`post-template.md`](../../blob/main/post-template.md)，把**全部內容**複製貼到你的新檔案，然後修改最上面兩條 `---` 之間的資料欄：

| 欄位 | 說明 |
|------|------|
| `title:` | 文章標題（如果標題裡有冒號 `:`，整個標題要用引號包起來："標題: 副標"）|
| `date:` | 發佈日期時間。**不能填未來的時間**，填了文章不會顯示 |
| `tags:` | 標籤，方括號內用逗號分隔：`[選書, 活動]`。**先看看既有文章用過哪些標籤，盡量沿用**，不要開太多新標籤 |
| `description:` | 一句話摘要。會顯示在文章列表、搜尋結果和電子報裡，**請務必填寫** |
| `image:` | 封面圖路徑（選填）。沒有封面就把這一整行刪掉 |

### 第 3 步：寫內文

兩條 `---` 之後就是內文，直接打字，記住這幾個符號就夠用：

```
## 段落標題（兩個井號＋空格）

**粗體文字**

[連結文字](https://網址)

![圖片說明](/assets/images/圖片檔名.jpg)

- 列表項目
```

**空一行才會分段**——只按一次 Enter 不會換段。

### 第 4 步：發佈

拉到頁面最下面（或右上角）按綠色的 **Commit changes** 按鈕。等 1–2 分鐘，到 [blog.in-common.tw](https://blog.in-common.tw) 重新整理就能看到。

## 上傳圖片

1. 打開 [`assets/images/` 資料夾](../../tree/main/assets/images)
2. 點 **Add file → Upload files**，把圖片拖進去，按 Commit changes
3. 圖片檔名請用**英文和數字**，並加上日期前綴避免撞名：
   - ✅ `2026-07-15-bookshelf.jpg`
   - ❌ `書架照片.jpg`、`IMG_1234.jpg`（容易跟別人撞名）
4. 在文章裡引用：`![圖片說明](/assets/images/2026-07-15-bookshelf.jpg)`

> 圖片請先壓縮過再上傳（手機原圖動輒好幾 MB，會拖慢網頁）。可以用 [Squoosh](https://squoosh.app/) 免費壓縮，寬度 1200px 以內就很夠了。

### 控制圖片寬度

預設圖片會以全寬置中顯示。想讓某張圖小一點，在圖片語法**後面**加上 `{: width="..." }`，
**高度會自動依原圖比例縮放**，不會變形：

| 寫法 | 效果 |
|------|------|
| `![說明](/assets/images/xxx.jpg){: width="60%" }` | 佔內文寬度的 **60%**（推薦，手機上會自動跟著縮）|
| `![說明](/assets/images/xxx.jpg){: width="400" }` | 固定 **400 像素**寬 |
| `![說明](/assets/images/xxx.jpg)` | 不加就是**全寬** |

注意事項：
- `{:` 前面**要有一個空格**、冒號後面**沒有**空格，整段緊接在 `)` 之後同一行
- 建議用**百分比**（如 `60%`）而不是固定像素——百分比在手機上會自動縮放，比較不會爆版
- 只影響那一張圖，其他圖不受影響

## 修改或下架文章

- **修改**：打開 `_posts/` 裡的那篇文章 → 點右上角鉛筆圖示 ✏️ → 改完 Commit changes
- **下架**：打開文章 → 點右上角「⋯」→ Delete file → Commit changes

---

## 出問題了？

**文章沒有出現：**
1. 檢查檔名是不是 `YYYY-MM-DD-name.md` 格式（最常見的錯誤）
2. 檢查 `date:` 是不是填了未來時間
3. 檢查最上面的 `---` 有沒有成對（第一行一條、資料欄結束一條）
4. 到 repo 的 **Actions** 頁籤看最新的 build 是綠勾還是紅叉——紅叉點進去看錯誤訊息

**標題顯示不完整或版面亂掉：**
- 標題或 description 裡有冒號 `:` 時，整段要用引號包起來：`title: "夏日選書: 五本消暑的書"`

有任何解決不了的問題，找工程師夥伴。
